Etimología

Simulación viene de raíces del latín relacionadas con semejanza o imitación. Modelo viene de una idea de medida, patrón o representación reducida. Una simulación no es la realidad misma. Es una imitación disciplinada para probar, estimar, explorar o visualizar un comportamiento que puede ser demasiado complejo, grande, pequeño, lento o peligroso para tratarlo directamente.

Un buen modelo es honesto sobre lo que deja fuera. Esa honestidad separa una simplificación útil de una animación engañosa.

Modelo, método, simulación

Un modelo define variables, supuestos, ecuaciones y fronteras. Un método numérico convierte esas ecuaciones en pasos computables. Una simulación ejecuta el modelo en el tiempo, en muestras, en agentes o en puntos de una red.

La distinción es crucial. Si el modelo está mal, un algoritmo perfecto solo produce más rápido el resultado equivocado. Si el método es inestable, incluso un buen modelo puede producir basura.

Experimento numérico

Modelar es más antiguo que las computadoras, pero la simulación computacional moderna se volvió poderosa en el siglo XX cuando métodos numéricos, computadoras electrónicas y muestreo estadístico empezaron a trabajar juntos. Física, clima, investigación nuclear, ingeniería, biología, economía y aprendizaje automático necesitaban estudiar sistemas que se resistían a soluciones exactas.

Verificación y validación

La simulación necesita disciplina porque una visualización atractiva puede ocultar fallas numéricas. Verificación pregunta si el código resuelve correctamente las ecuaciones. Validación pregunta si las ecuaciones representan el sistema real con suficiente fidelidad para la afirmación que se hace. La cuantificación de incertidumbre mide cómo los supuestos, parámetros y errores numéricos afectan el resultado.

Caos y simulación estocástica

La simulación no solo trata de predicción suave. Lorenz mostró que ecuaciones deterministas pueden ser extremadamente sensibles a las condiciones iniciales. La simulación estocástica maneja sistemas donde el azar forma parte del modelo, no solo del ruido numérico.

La simulación es poderosa y peligrosa. Puede revelar patrones ocultos al cálculo directo, pero también puede crear falsa confianza cuando los supuestos no se declaran con claridad.

Supuestos y límites

Todo modelo tiene una frontera. Nubes de probabilidad orbital, redes de difusión, atractores caóticos, autómatas celulares, movimiento de agentes, dinámica molecular y muestreo de Monte Carlo enseñan cosas distintas porque simplifican la realidad de formas distintas.

El modelo orbital de hidrógeno forma parte de esta categoría más amplia. No es una trayectoria literal del electrón. Es una estructura de probabilidad muestreada, así que el movimiento necesita explicación y no solo atractivo visual.

Frontera Moderna

La simulación moderna incluye cuantificación de incertidumbre, modelos sustitutos, simulación diferenciable, gemelos digitales, cómputo de alto rendimiento, métodos Monte Carlo, modelado por elementos finitos, conjuntos climáticos y aprendizaje automático informado por física. La frontera no es solo hacer que una simulación parezca realista; es saber cuándo el resultado es confiable.

Una simulación seria reporta supuestos, condiciones de frontera, discretización, convergencia, estabilidad, validación y error. Sin eso, una visualización hermosa puede engañar. Con eso, la simulación se vuelve compañera experimental, no solo animación.

Glosario de Términos Técnicos

  • ▣ODE: Ecuación diferencial ordinaria, donde la función desconocida depende de una sola variable independiente, normalmente el tiempo.
  • ▣PDE: Ecuación diferencial parcial, donde el campo desconocido depende de varias variables como espacio y tiempo.
  • ▣Condición de frontera: Una regla impuesta en el borde de un dominio, como comportamiento aislado, fijo o envolvente en la red de calor.
  • ▣Condición inicial: El estado de partida desde el que comienza la evolución numérica.
  • ▣Simulación estocástica: Una simulación donde la aleatoriedad forma parte del modelo y no solo del error numérico.
