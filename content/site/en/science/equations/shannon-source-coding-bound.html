Shannon Source Coding Bound

Equation

$$ H(X)\le L < H(X)+1 $$

Literal Reading

Read it as: Shannon entropy of \(X\) is less than or equal to the expected codeword length \(L\), and \(L\) is less than Shannon entropy plus one bit.

Common Reading

An optimal prefix code can get the average bits per symbol within one bit of entropy.

Symbols

\(H(X)\): Shannon entropy of the source.

\(L\): expected number of bits per symbol used by a prefix code.

\(+1\): the finite-symbol overhead from using whole bits in a practical code.

Meaning of the Equation

This is a source-coding limit for expected length, not a statement about every individual message. Under the usual prefix-code assumptions for a discrete memoryless source, entropy sets the average compression target: frequent symbols can receive shorter codewords, rare symbols receive longer ones, and the best average cannot beat \(H(X)\). For long block codes, the matching idea uses entropy rate instead of a single-symbol entropy.
